## 適用時機

當使用者說「幫我建立一個技能」、「新增 skill」、「製作技能」、「我想要一個可以 XXX 的技能」，
或說「把我剛才做的事情變成一個技能」時啟用。

---

## 第一步：捕捉意圖

**優先從當前對話歷史中提取答案**，使用者可能已經示範了一個完整的工作流程。
若對話中已有足夠資訊（使用的工具、操作順序、使用者修正過的步驟、輸入/輸出格式），
直接提取並讓使用者確認，而不是重新詢問。

若資訊不足，使用 `ask_user_question` 工具一次性蒐集以下資訊。
**每個問題都應盡量提供選項**，並在推薦選項後標註「(推薦)」：

**問題 1：這個技能要讓 Agent 做什麼？**
**問題 2：什麼情況下觸發？**
**問題 3：預期的輸出格式是什麼？**
提供選項，例如：
- 「純文字回覆 (推薦)」、「寫入 artifacts/ 的 .md 檔案」、「HTML 視覺化」、「結構化 JSON」

**取得確認後再進行下一步。**

---

## 第二步：起草 SKILL.md

根據蒐集到的資訊起草完整的 SKILL.md，包含：

- 清楚的觸發時機描述（供 LLM 判斷何時 activate）
- 具體的操作步驟（足夠精確，讓 agent 不需猜測）
- 若有參考文件需求，在 body 中說明「參考文件位於 `references/` 目錄，在 XXX 情況下閱讀 `references/YYYYY.md`」
- SKILL.md 內容應控制在 **500 行以內**；若接近此限制，拆分至 `references/` 子文件，並在主文件中加入明確的「何時閱讀哪個檔案」指引

---

## 第三步：寫入與驗證

呼叫 `write_file`：
- `path`: `skills/{name}/SKILL.md`
- `content`: 完整的 SKILL.md 內容（含 frontmatter）

**處理系統回傳的驗證結果：**
- **驗證失敗**：根據錯誤訊息自行修正，重新呼叫 `write_file`，無需詢問使用者
- **驗證成功**：告知使用者技能已建立，本次對話可立即用 `activate_skill('{name}')` 啟用，下次對話起自動出現在技能清單

---

## 第四步：補充資源檔案

若技能需要腳本或詳細參考文件，繼續用 `write_file` 寫入：

```
skills/{name}/
├── SKILL.md                    ← 主文件（≤ 500 行）
├── scripts/
│   └── process.py              ← 可執行腳本
├── references/
│   └── REFERENCE.md            ← 詳細參考文件（超過 300 行需加目錄）
└── assets/
    └── template.txt            ← 模板與靜態資源
```

**參考文件超過 300 行時，在文件開頭加入目錄**，列出各節標題與行號，方便模型快速定位。

## SKILL.md 格式規範

```
---
name: skill-name               # 必填。小寫 kebab-case，1–64 字元，只含字母/數字/連字符
description: 說明功能與觸發時機  # 必填。1–1024 字元，清楚說明何時啟用
license: MIT                   # 選填
metadata:                      # 選填，鍵值對
  author: system
  version: "1.0"
---

# 技能標題

## 適用時機
說明何時啟用（使用者說什麼話、在什麼情境下）。

## 操作步驟
具體的指令與操作流程，足夠精確讓 agent 不需猜測。

## 參考文件（若有）
- 在 XXX 情況下，閱讀 `references/detail.md` 了解完整規則
- 在 YYY 情況下，閱讀 `references/examples.md` 查看範例
```

---

## 名稱命名規則

- 全小寫英文
- 只允許字母（a-z）、數字（0-9）、連字符（`-`）
- 不能以連字符開頭或結尾
- 不能有連續連字符（`--`）
- 最多 64 字元
- **技能目錄名稱必須與 `name` 欄位完全一致**（系統自動處理）

---

## 注意事項

- 驗證由系統自動執行，不需要呼叫額外工具
- 驗證失敗時，根據錯誤訊息修正後直接重新 `write_file`，系統會再次驗證
- 技能資料夾下可有多個子目錄（scripts/、references/、assets/），一律用 `write_file` 寫入
- 如需查看技能是否已存在，可呼叫 `activate_skill('{name}')` 確認
